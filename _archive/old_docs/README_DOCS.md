## 📊 システム概要

### 現在のシステム構成（v2.0 再構築版）

- **フロントエンド**: Vite + React 18 + TypeScript + Tailwind CSS
- **ホスティング**: Firebase Hosting（CDN・SSL・カスタムドメイン）
- **バックエンド**: Cloud Functions v2（Node.js 20 + TypeScript）
- **データベース**: Cloud Firestore
- **認証**: Firebase Authentication（管理者のみ / Googleログイン）
- **メール通知**: Resend API（無料枠 3,000通/月）
- **CI/CD**: GitHub Actions

### 主要機能

1. **予約フォーム**: 施設・日時・利用者情報の入力
2. **重複チェック**: 既存予約との重複確認
3. **データ管理**: スプレッドシートへの予約データ保存
4. **カレンダー連携**: Googleカレンダーへの予約登録
5. **エラー処理**: バリデーション・エラーハンドリング

### 既知の制約・課題（v1 GAS版 → 再構築で解消済み）

- **GAS複数アカウント問題**: ✅ 解消（Firebase Hostingに移行）
- **スケーラビリティ**: ✅ 解消（Cloud Functions + Firestoreに移行）
- **テスト不可**: ✅ 解消（Jest + Firebase Emulatorで全層テスト可能）
- **型安全なし**: ✅ 解消（TypeScript + zodで全層統一）
- **デプロイ属人化**: ✅ 解消（GitHub Actionsで自動化）
